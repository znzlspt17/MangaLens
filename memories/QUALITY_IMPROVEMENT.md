## 2. 개선 방안

### A. 번역 품질 개선

#### A-1. 문맥 윈도우 번역 ⭐ 최대 효과

- **현재**: 말풍선 21개를 각각 독립 번역 (1:1)
- **개선**: 같은 페이지의 전체 대사를 reading order 순으로 합쳐 한번에 번역 후 분리
- **효과**: 대화 흐름 이해 → "家系ラーメン"과 "イエケイ?" 연결 가능
- **구현 방식**:
  ```
  [System] You are a manga translator...
  [User]
  <1> 今日のお
  <2> それが大人だった
  ...
  <21> 私はラーメン食べれませんでした
  ```
  → 모델이 `<1>` ~ `<21>` 태그 단위로 분리 출력
- **위험**: 파싱 실패 시 개별 번역 fallback 필요
- **비용**: 토큰 수 증가로 추론 시간 ~2배
- **난이도**: ★★★☆☆
- **상태**: ✅ 완료 — 페이지 전체 대사를 `<1>`~`<N>` 태그로 합쳐 일괄 번역, 70% 파싱 임계치, 실패 시 개별 번역 fallback

#### A-2. 시스템 프롬프트 강화

- **현재**: 단순 "Translate JA → KO" 지시
- **개선 내용**:
  - 만화 대사 특화: 구어체, 간사이 방언(〜やん, 〜とんねん), 슬랭 지원
  - 문화 고유명사 음차 규칙: 人名은 한국 관용 표기(キアヌ→키아누)
  - 의성어/의태어 한국어 대응 지시
  - 격식체/반말 일관성 유지 지시
- **난이도**: ★☆☆☆☆
- **상태**: ✅ 완료

#### A-3. OCR 후처리 보정

- **현재**: manga-ocr 결과를 그대로 번역기에 전달
- **개선 내용**:
  - 카타카나 장음 보정: `オマッセシマシター` → `お待たせしました`
  - 일본어 사전 기반 spell check (MeCab 형태소 분석)
  - confidence 기반: 저신뢰 OCR 결과에 "might be unclear" 힌트 부여
- **난이도**: ★★★☆☆
- **상태**: 미착수

#### A-4. 번역 후처리 강화

- **현재**: `_postprocess()`가 노이즈 제거만 수행
- **개선 내용**:
  - 경어체 일관성 체크: 같은 캐릭터 반말/존댓말 혼용 방지
  - 의미 없는 음차 감지: 카타카나→한국에 알려진 고유명사 사전 매칭
  - 길이 비율 검증 강화: JA→KO 비율 1:1~1:1.5, 현재 6배 허용은 과도
- **난이도**: ★★☆☆☆
- **상태**: 미착수

#### A-5. Few-shot 예시 삽입

- **현재**: zero-shot 번역
- **개선**: 시스템 프롬프트에 만화 번역 예시 3~5쌍 포함
  ```
  例: "お待たせしましたー" → "기다리게 해서 죄송합니다~"
  例: "家系ラーメン" → "이에케이 라면"
  例: "はまっとんねん" → "빠져있거든"
  ```
- **토큰 비용**: ~100토큰 추가 (미미)
- **난이도**: ★☆☆☆☆
- **상태**: ✅ 완료

### B. 텍스트 렌더링 품질 개선

#### B-1. 폰트 업그레이드

- **현재**: `NotoSansKR-Regular.ttf` 1종
- **개선안**:
  - **NotoSansKR-Bold.ttf** 추가: 만화 대사는 Bold가 기본
  - **만화 전용 폰트** 도입: Nanum Pen Script, KCC-Ganpan, 마루부리 등
  - 또는 **NotoSansKR[wght].ttf** Variable 폰트 1개로 Regular~Bold 커버
- **효과**: 만화 대사 느낌 대폭 상승
- **난이도**: ★☆☆☆☆
- **상태**: ✅ 완료 — `NotoSansKR-VariableFont_wght.ttf` 다운로드, wght 축 100~900, Bold 700 자동 적용

#### B-2. 한국어 단어 단위 줄바꿈

- **현재**: `_wrap_text()` 글자 단위 줄바꿈
- **개선**: 공백/조사 경계에서 우선 줄바꿈, 불가능 시에만 글자 단위
  ```
  현재: "죄송합니다만, 동석해도 괜찮\n을까요?"  ← 단어 중간 절단
  개선: "죄송합니다만,\n동석해도 괜찮을까요?"    ← 단어 단위
  ```
- **난이도**: ★★☆☆☆
- **상태**: ✅ 완료 — `_split_words()` + 공백/구두점 경계 우선 줄바꿈, 단어 초과 시 글자 단위 fallback

#### B-3. 적응형 스트로크/크기

- **현재**: `STROKE_WIDTH=2` 고정, `font_size * 0.85` 고정
- **개선**:
  - 스트로크 폰트 크기 비례: `max(1, font_size // 20)`
  - 85% 축소 대신 bbox 면적 대비 텍스트 면적 비율 기반 동적 조절
  - 소형 버블(w<60 or h<60)은 축소 없이 피팅
- **난이도**: ★★☆☆☆
- **상태**: ✅ 완료 — `max(1, font_size // 20)` + 소형 버블 축소 제외

#### B-4. 줄간격 최적화

- **현재**: `_LINE_HEIGHT_RATIO=1.4`
- **개선**: 한국어 가로쓰기는 `1.2~1.3` 최적
  - 1~2줄: `1.3`
  - 3줄 이상: `1.2` (공간 절약)
- **난이도**: ★☆☆☆☆
- **상태**: ✅ 완료 — `_LINE_HEIGHT_RATIO_FEW=1.3` / `_LINE_HEIGHT_RATIO_MANY=1.2`

#### B-5. 말풍선 배경 기반 자동 색상

- **현재**: 무조건 검정 텍스트 + 흰 윤곽
- **개선**: inpainted bubble 영역 평균 밝기 분석
  - 밝은 배경(>180): 검정 글씨 + 흰 윤곽 (현재)
  - 어두운 배경(<80): 흰 글씨 + 검정 윤곽
  - 중간: 적응형
- **난이도**: ★★☆☆☆
- **상태**: 미착수

#### B-6. 줄바꿈 시 좌측 글자 잘림 방지 ✅ 완료

- **현재(수정 전)**: `_wrap_text()`가 `usable_w` 기준으로 줄 길이를 결정하나, 실제 그릴 때 stroke가 왼쪽으로 `stroke_width`만큼 튀어나와 오버레이 경계 밖에서 잘림
- **수정 내용**:
  - `wrap_w = usable_w - 2 × stroke_width` 로 줄바꿈 기준 너비를 스트로크 여유분만큼 축소
  - `_draw_horizontal()`에서 `x = max(stroke_width, _PADDING + …)` 하한 적용
- **파일**: `server/pipeline/text_renderer.py`
- **난이도**: ★☆☆☆☆
- **상태**: ✅ 완료 (Phase 15, D-022)
